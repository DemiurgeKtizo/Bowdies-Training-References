# Bowdie's Training References — Engine Conventions

## Pairing notes — preservation vs. correction

`pairing-notes.js` contains 50,646 entries. Two categories:

1. **Engine-templated (~33K notes):** generated by `pairing_engine_generator.js`. Identifiable by `isTemplatedNote()` signature regexes (`runs straight into ... — the`, `Gold standard; the {TAG} lock that sells itself`, `Avoid; Reach for`, etc.). These are rewritten on every regen pass. Never hand-edit them — the next regen will overwrite.

2. **Hand-written editorial (~17K notes):** preserved by default. The engine sees them via `isTemplatedNote() === false` and skips regeneration.

### Editorial preservation — and when to break it

Hand-written editorial is preserved as a rule. The exception:

> **Editorial CAN be edited when it contains a factual error about menu preparation, ingredients, or plating.**

Floor-grade accuracy beats preservation. Examples of legitimate edits:

- Cooking method drift (e.g., a note says "the steak's seared fat" but Bowdie's steaks are flame-grilled, rested, and oven-finished — never pan-seared)
- Ingredient/preparation mismatches ("the dish includes truffle oil" when the kitchen doesn't use truffle oil)
- Outdated bottle facts (proof, age statement, distillery — when the bottle changes)

Edits should be applied via a script (e.g., `fix_cooking_method_in_editorial.js`) that pattern-matches the specific factual error and replaces it cleanly. Never blanket-edit editorial — match the specific phrase, replace with the correct phrase, log the count.

### Cooking method reference

Bowdie's prep methods (use these terms when writing notes):

- **Steaks** (Tomahawk, Cowboy Ribeye, Kansas City, Filet, Porterhouse, Bone-In Filet) — flame-grilled, rested, oven-finished. Use language like "flame-grilled char", "grilled crust", "char-and-fat", "rendered fat". **Never** "seared crust", "seared fat", "pan-seared".
- **Scallops** — seared. "Caramelized sear" is correct.
- **Yellowfin Tuna (Tuxedo-Crusted)** — seared rare. "Seared rare crust" is correct.
- **Salmon (Faroe Island)** — pan-finished. "Rich oily flesh" is the texture anchor.
- **Chicken (Roast Half Chicken)** — roasted. "Herbed crisp skin" is the texture anchor. Preparation may vary seasonally.

When in doubt about prep, do not invent details — leave it general or check with the kitchen.

## Regen pipeline

1. `node engine_health_check.js` — surface blockers before regen
2. `node engine_snapshot_test.js` — verify 81 anchor pairs unchanged  
3. `node regenerate_templated_notes.js` — rewrite templated notes
4. (If editorial needs factual correction) write a `fix_*_in_editorial.js` script, run it
5. `node engine_snapshot_test.js --update` to lock in intended changes

## Class taxonomy (drinks)

13 classes in current vocabulary:

- `BIG_RED` — heavy/big red wines (Caymus, Opus, Silver Oak)
- `ELEGANT_RED` — medium-bodied reds, Pinots, Chianti, Bordeaux blends
- `BOURBON_BOLD` — all whiskey-family (bourbon/scotch/rye/Irish/Japanese)
- `TEQUILA_BOLD` — añejo / extra-añejo / heavier reposado (per `class_drinks.json` tag)
- `MEZCAL` — all mezcals
- `COGNAC` — non-luxury cognac (Hennessy, Courvoisier)
- `COGNAC_LUXURY` — luxury-icon cognac (Louis XIII)
- `SPARKLING` — Champagne, Cava, Prosecco
- `WHITE_WINE` — still whites
- `GIN` — all gins
- `VODKA` — all vodkas
- `LIGHT_SPIRIT` — light rum, blanco tequila
- `HEAVY_SPIRIT` — heavy non-classified spirits (rare fallback)
- `COCKTAIL_BOLD` — whiskey-forward cocktails
- `COCKTAIL_LIGHT` — citrus / gin / tequila cocktails (light register)
- `SWEET_LIQUEUR` — non-bitter sweet liqueurs (Sambuca, Frangelico, Limoncello)
- `APERITIVO_BITTER` — bitter amari, Campari, Aperol, Fernet, vermouths
- `SWEET_WINE` — Port, Sauternes, dessert wines

When adding a new bottle, ensure it's classified correctly. Run `engine_health_check.js` after to verify no new DEFAULT-bucket entities.
